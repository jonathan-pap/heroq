# Art index (`assets/`)

> **Purpose:** Indexes every PNG / JPG subfolder, the naming
> conventions used in each, and the steps to add new art (furniture,
> tokens, tiles, textures).
>
> **Related:**
> [`PROJECT_STRUCTURE.md`](../PROJECT_STRUCTURE.md) (top-level map),
> [`data/SCHEMAS.md`](../data/SCHEMAS.md) (`canonical-pieces.yaml`
> resolves filenames here),
> [`public/FRONTEND.md`](../public/FRONTEND.md) (loaders that read
> these PNGs).

PNG / JPG art served statically by [`server.js`](../server.js) under
`/assets/...`. No build step â€” what's on disk is what the browser
fetches.

## Subfolders

| Folder | Content | Naming |
|---|---|---|
| `board/` | Source board scans + reference photos. Only `board_v2.png` is read at runtime (by the builder and the floor-texture extractor). Everything else is for offline scripts. | `board_v2.png`, `board3.png` (red-mask), `board2.png`, `board.jpg`. |
| `room_textures/` | Hand-prepared per-room floor textures. The active source for the game's floor-texture overlay. | `room_NN.png` (per room id, 01-22). `corridor.png` + `corridor_no_walls.png` (toggle variants). |
| `floors/` | Legacy floor-extraction output from `scripts/extract-room-floors.js`. Read by the builder as a reference photo. | `playable.png`, `r<NN>.png`, `_index.json` (cellW/cellH metadata). |
| `furniture/` | Canonical heroscribe furniture icons + alt-art variants. Referenced by `canonical-pieces.yaml`. | Canonical: `Bookcase.png`, `Tomb.png`, `AlchemistsBench.png`, â€¦ Alt: `Bookcase-1x3.png`, `Tomb-2x3.png`, `Alchemist Bench-2x3.png` (spaces in filenames OK). |
| `monsters/` | Monster art. The runtime uses `<Type>-Token.png` (round tokens that match the printed pieces). `<Type>-Card.png` is the bigger card art used in card-overlay UIs. | `Goblin-Token.png`, `Orc-Token.png`, `Mummy-Card.png`, plus expansion sets like `TheFrozenHorror.*`. |
| `heros/` | Hero portraits + variant tokens (Male / Female). | `<Hero>-Male-Token.png`, `<Hero>-Female-Token.png`, `<Hero>-Male-Card.png`, plus legacy single-variant `<Hero>.png`. |
| `tiles/` | Door / stair / trap / rubble icons + letter & number tiles for canonical board labelling. | `SingleBlockedSquare.png`, `DoubleBlockedSquare.png`, `PitTrap.png`, `SpearTrap.png`, `Stairway.png`, `LetterA.png` â€¦ `Number12.png`. |
| `cards/` | Card-back art for the spell / treasure / artifact / equipment decks. | `card_backs/<Element> Spell.png`, `card_backs/Treasure.png`, etc. |
| `map_qa/` | QA preview PNGs generated by `scripts/render-quest-maps.js`. | `quest<N>-<slug>.png`. |
| `maps/` | Source HeroScribe XML files for canonical quests. Input for `scripts/build-quest1-from-xml.js`. | `HQBase-NN-<name>_US.xml`. |

## How code references these

- **Furniture filenames** â†’ `data/canonical-pieces.yaml`
  (`file`, `altFile`, optional `dir`). Don't hardcode in JS â€” the
  three frontends fetch the YAML and resolve through it.
- **Monster tokens** â†’ `MONSTER_TYPE_FILE` in `client.js` +
  `map-editor.js` (still hardcoded â€” backlog item).
- **Hero tokens** â†’ `HERO_FILE` + `HERO_NAMES` (same).
- **Tile icons** â†’ `TILE_FILE` (same).
- **Floor textures** â†’ `client.js` `loadRoomTexture()` /
  `loadCorridorTexture()`; paths built from `room_NN.png` /
  `corridor*.png` directly.
- **Reference photo** â†’ `builder.js` loads
  `/assets/board/board_v2.png` for the cross-check overlay.

## Adding new art

| Want to add a â€¦ | Drop the PNG into â€¦ | Then |
|---|---|---|
| New furniture piece | `assets/furniture/` | Add an entry to `data/canonical-pieces.yaml` with `file:` (and `altFile:` if applicable). |
| Alt-art variant of an existing piece | `assets/furniture/` (any filename) | Set `altFile:` on the piece's YAML entry. |
| New monster token | `assets/monsters/<Type>-Token.png` | Add to `MONSTER_TYPE_FILE` in `client.js` + `map-editor.js` (until the monster YAML consolidation lands â€” see `docs/BACKLOG.md`). |
| New hero variant token | `assets/heros/<Hero>-<Variant>-Token.png` | The `loadAllSprites()` loop already covers `Male` / `Female`. New variants need a tiny constant update. |
| New trap / tile | `assets/tiles/` | Add to `TILE_FILE` in `client.js`. |
| Replacement room-floor texture | `assets/room_textures/room_NN.png` | Bump cache buster `?v=N` in `client.js`'s loader so browsers refetch. |
